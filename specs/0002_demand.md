# 合同管理后台需求文档

## 1. 文档目标

本文档描述本地后台管理应用第一版需要实现的业务功能、页面范围、数据字段和交互规则。

第一版重点实现：

- 登录页面
- 合同管理页面
- 合同新增、编辑、查看详情、删除
- 增补合同新增、编辑、查看详情、删除
- 检测报告管理菜单占位

本文档不包含具体代码实现。

## 2. 产品定位

应用是一个面向内部使用的后台管理程序，用于记录合同、合同附件、收款、保证金、业务员提成以及增补合同信息。

系统第一版只做本地单机使用，不做云端同步、不做多人协作、不做线上后端服务。

## 3. 用户与登录

### 3.1 登录入口

打开应用后，如果用户未登录，必须先进入登录页面。

### 3.2 登录账号

系统第一版只有一个固定账号：

| 字段 | 值 |
| --- | --- |
| 账号 | chenglu |
| 密码 | 88888888 |

### 3.3 登录成功后跳转

登录成功后默认进入“合同管理”页面。

### 3.4 登录失败提示

账号或密码错误时，需要在登录页面提示用户登录失败，不进入后台页面。

## 4. 页面范围

### 4.1 后台布局

登录成功后进入后台主界面。

后台左侧菜单包含 2 个一级菜单：

| 菜单 | 第一版状态 |
| --- | --- |

## 5. 合同管理列表

### 5.1 页面组成

合同管理页面主体是一个合同列表表格。

表格上方需要有“添加合同”按钮。

### 5.2 合同列表字段

合同列表表格需要展示以下字段：

| 字段 | 说明 |
| --- | --- |
| 时间 | 合同日期或合同记录时间 |
| 项目名称 | 合同对应项目名称 |
| 业主单位 | 合同业主单位 |
| 合同金额 | 初始合同金额，不包含增补合同金额 |
| 履约保证金 | 是否有履约保证金及保证金金额 |
| 履约保证金形式 | 例如现金、保函等 |
| 履约保证金退还时间 | 约定的退还时间，不代表实际已经退还 |
| 质保金 | 是否有质保金及质保金金额 |
| 质保金形式 | 例如现金、扣款等 |
| 质保金退还时间 | 约定的退还时间，不代表实际已经退还 |
| 合同附件 | 合同附件入口或附件数量 |
| 已收款金额 | 初始合同已收款金额合计 |
| 未收款金额 | 初始合同金额减去初始合同已收款金额 |
| 操作 | 编辑、查看详情、删除、增补合同 |

### 5.3 合同列表操作

每一行合同需要提供以下操作：

| 操作 | 说明 |
| --- | --- |
| 编辑 | 打开合同编辑界面，允许修改合同信息 |
| 查看详情 | 打开合同详情界面，只读展示合同完整信息 |
| 删除 | 删除当前合同，需要二次确认 |
| 增补合同 | 给当前合同添加一条增补合同 |

### 5.4 合同展开子表格

合同列表需要支持展开行。

展开后展示当前合同下的增补合同列表。

## 6. 合同详情字段

一个合同详情包含以下信息：

| 模块 | 字段 |
| --- | --- |
| 基础信息 | 时间、项目名称、业主单位、合同金额、合同附件 |
| 经办人 | 经办人姓名、经办人电话、经办人职位 |
| 履约保证金 | 是否有履约保证金、履约保证金金额、履约保证金形式、约定退还时间、是否已经退还 |
| 质保金 | 是否有质保金、质保金金额、质保金形式、约定退还时间、是否已经退还 |
| 收款记录 | 收款金额、收款日期 |
| 业务员提成 | 业务员、提成、提成付款时间 |
| 增补合同 | 增加合同金额、增补合同日期、增补合同附件、增补合同收款金额、增补合同收款日期 |

## 7. 添加和编辑合同

### 7.1 基础信息

添加或编辑合同时，需要填写：

| 字段 | 要求 |
| --- | --- |
| 时间 | 必填 |
| 项目名称 | 必填 |
| 业主单位 | 必填 |
| 合同金额 | 必填，表示初始合同金额，不包括后续增补合同金额 |
| 合同附件 | 可上传附件 |

### 7.2 经办人

经办人可以有多个。

添加或编辑合同时，需要提供一个可编辑表格，用于新增、修改、删除经办人。

经办人表格字段：

| 字段 | 说明 |
| --- | --- |
| 经办人姓名 | 经办人姓名 |
| 经办人电话 | 经办人联系电话 |
| 经办人职位 | 经办人在业主单位的职位 |

### 7.3 履约保证金

履约保证金不一定存在。

表单中需要有“是否有履约保证金”开关。

开关关闭时，不需要填写履约保证金相关字段。

开关开启时，需要填写：

| 字段 | 说明 |
| --- | --- |
| 履约保证金 | 履约保证金金额 |
| 履约保证金形式 | 保证金形式，例如现金、保函等 |
| 履约保证金退还时间 | 约定的退还时间，不代表实际已经退还 |
| 是否已经退还 | 选择已退还或未退还 |

### 7.4 质保金

质保金不一定存在。

表单中需要有“是否有质保金”开关。

开关关闭时，不需要填写质保金相关字段。

开关开启时，需要填写：

| 字段 | 说明 |
| --- | --- |
| 质保金 | 质保金金额 |
| 质保金形式 | 质保金形式，例如现金、扣款等 |
| 质保金退还时间 | 约定的退还时间，不代表实际已经退还 |
| 是否已经退还 | 选择已退还或未退还 |

### 7.5 收款记录

一个合同可以有多条收款记录。

添加或编辑合同时，需要提供一个可编辑表格，用于新增、修改、删除收款记录。

收款记录表格字段：

| 字段 | 说明 |
| --- | --- |
| 收款金额 | 本次收款金额 |
| 收款日期 | 本次收款日期 |

合同已收款金额为所有收款记录的金额合计。

合同未收款金额为：

```text
合同金额 - 已收款金额
```

合同金额只代表初始合同金额，不包含增补合同金额。

### 7.6 业务员提成

一个合同可以有多个业务员提成记录。

添加或编辑合同时，需要提供一个可编辑表格，用于新增、修改、删除业务员提成记录。

业务员提成表格字段：

| 字段 | 说明 |
| --- | --- |
| 业务员 | 业务员姓名 |
| 提成 | 提成金额 |
| 提成付款时间 | 提成实际付款时间 |

## 8. 增补合同管理

### 8.1 增补合同入口

在合同列表每一行的操作栏中，需要提供“增补合同”按钮。

点击后为当前合同添加一条增补合同。

### 8.2 增补合同字段

添加增补合同时，需要填写：

| 字段 | 要求 |
| --- | --- |
| 增加合同金额 | 必填，表示本次增补增加的金额 |
| 增补合同日期 | 必填 |
| 增补合同附件 | 可上传附件 |
| 增补合同收款记录 | 可维护多条收款记录 |

### 8.3 增补合同收款记录

一个增补合同可以有多条收款记录。

收款记录需要使用可编辑表格维护。

增补合同收款记录字段：

| 字段 | 说明 |
| --- | --- |
| 收款金额 | 本次增补合同收款金额 |
| 收款日期 | 本次增补合同收款日期 |

增补合同已收款金额为该增补合同所有收款记录的金额合计。

增补合同未收款金额为：

```text
增加合同金额 - 增补合同已收款金额
```

### 8.4 增补合同列表字段

合同列表展开后的增补合同表格只需要展示：

| 字段 | 说明 |
| --- | --- |
| 增加合同金额 | 本次增补增加的金额 |
| 增补合同日期 | 增补合同日期 |
| 增补合同附件 | 附件入口或附件数量 |
| 增补合同收款金额 | 增补合同已收款金额合计 |
| 未收款金额 | 增加合同金额减去增补合同已收款金额 |
| 操作 | 编辑、查看详情、删除 |

### 8.5 增补合同操作

每一条增补合同需要提供以下操作：

| 操作 | 说明 |
| --- | --- |
| 编辑 | 修改增补合同信息 |
| 查看详情 | 只读查看增补合同完整信息 |
| 删除 | 删除增补合同，需要二次确认 |

## 9. 附件需求

### 9.1 合同附件

合同支持添加合同附件。

附件用于保存合同文件、扫描件或其他相关材料。

### 9.2 增补合同附件

增补合同支持添加增补合同附件。

### 9.3 附件操作

附件需要支持：

| 操作 | 说明 |
| --- | --- |
| 添加附件 | 选择本地文件并保存到应用数据目录 |
| 查看或打开附件 | 用系统默认程序打开附件 |
| 删除附件 | 从当前业务记录移除附件 |

## 10. 删除规则

删除合同或增补合同时，需要二次确认。

删除合同后，该合同下的经办人、收款记录、业务员提成、增补合同和附件不再展示。

删除增补合同后，该增补合同下的收款记录和附件不再展示。

第一版建议使用软删除，避免误删后无法恢复。

## 11. 金额计算规则

### 11.1 初始合同金额

合同金额表示初始合同的金额，不包含任何增补合同金额。

### 11.2 合同已收款金额

合同已收款金额只统计初始合同自身的收款记录。

```text
合同已收款金额 = 初始合同收款记录金额合计
```

### 11.3 合同未收款金额

合同未收款金额只针对初始合同金额计算。

```text
合同未收款金额 = 合同金额 - 合同已收款金额
```

### 11.4 增补合同已收款金额

```text
增补合同已收款金额 = 当前增补合同收款记录金额合计
```

### 11.5 增补合同未收款金额

```text
增补合同未收款金额 = 增加合同金额 - 增补合同已收款金额
```

## 12. 表单校验需求

### 12.1 必填字段

合同必填字段：

- 时间
- 项目名称
- 业主单位
- 合同金额

增补合同必填字段：

- 增加合同金额
- 增补合同日期

### 12.2 金额字段

金额字段必须是数字，且不能小于 0。

金额字段包括：

- 合同金额
- 履约保证金
- 质保金
- 收款金额
- 增加合同金额
- 增补合同收款金额
- 提成

### 12.3 日期字段

日期字段需要使用日期选择器，不建议让用户手工输入任意文本。

### 12.4 保证金字段联动

履约保证金或质保金开关关闭时，对应金额、形式、约定退还时间、是否退还不参与必填校验。

履约保证金或质保金开关开启时，需要填写对应的约定退还时间。约定退还时间不与“是否已经退还”联动；无论已退还还是未退还，都表示合同约定的退还日期。

## 13. 第一版不做的内容

第一版明确不做：

- 检测报告管理业务功能
- 多用户账号管理
- 权限管理
- 云端同步
- 线上后端服务
- 移动端应用
- 合同审批流程
- 合同到期提醒
- 自动生成合同编号
- 财务系统对接
